# E-Commerce AI Copilot

基于大语言模型（LLM）与 Agentic Workflow 构建的电商数据运营与风控工作台。

本项目并非一个简单的“对话机器人”，而是一个以**数据准确性**和**业务风控**为核心的自动化执行流（Agent）。通过引入指标语义层（Metric Store）与动态熔断机制，彻底剥离了 LLM 在处理精确数值时产生的幻觉，实现“自然语言查询 -> 数据沙箱计算 -> 阈值风控阻断 -> SOP 知识库联动”的闭环业务链路。

## 🎯 业务痛点与解决方案

在真实的电商数据中台中，直接让 LLM 编写 SQL 或 Pandas 代码往往面临极高的稳定性风险（脏数据注入、时序错位、计算幻觉）。
本项目采用了 **“调度大脑与执行组件解耦”** 的架构：
* **大模型仅负责意图路由与任务拆解 (Planner/Reasoner)**
* **所有业务指标计算由底层硬编码的 Python 沙箱严格接管 (Executor)**

## ⚙️ 核心架构与特性

### 1. 指标语义层与防御性计算 (Semantic Metric Store)
摒弃了脆弱的 Text-to-Pandas 方案，系统底层构建了企业级的指标配置字典。
* 支持多维数据查询（GMV走势、热销榜单、退款异常商品、VIP流失画像）。
* **高鲁棒性**：面对未注册的数据维度查询，系统会执行拦截指令而非强行伪造数据。

### 2. 动态风控与熔断机制 (Data Circuit Breaker)
具备工业级的数据质量校验与业务预警能力：
* **时序漂移监控**：内置数据时间戳基线校验，拦截陈旧历史数据注入（防止 GIGO）。
* **阈值熔断**：实时监控大盘数据（如退货率突破 8% 或客单价低于 50 元），触发异常后强制阻断常规对话，抛出红色警报。

### 3. 企业内部 SOP 向量检索 (RAG)
集成 `sentence-transformers` 与 `FAISS` 向量数据库。
* 当触发数据熔断时，系统会自动在知识库中检索对应的应急预案（如《高退货率异常处置 SOP》）。
* 实现“发现问题 -> 告警 -> 给出标准化实操步骤”的自动化决策。

### 4. 全链路执行追踪 (Traceable UI)
基于 Streamlit 构建的前端工作台。
* 摒弃黑盒输出，将 Agent 的“思考链 (Thought)”、“算子调用 (Tool Use)”、“响应耗时 (Latency)”与“载荷快照 (Payload)”全流程结构化展示。
* 原生支持动态折线图渲染与数据报表输出。
